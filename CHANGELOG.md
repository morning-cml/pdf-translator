# 变更记录（Changelog）

本文件记录每一步优化与各版本之间的区别。遵循[语义化版本](https://semver.org/lang/zh-CN/)。
每条 git 提交是最小记录单元；下方按**里程碑版本**归并，写清"本版相比上一版新增了什么"。

> 说明：项目 2026-07-19 才引入 git，之前有 8 轮迭代（核心引擎）作为「初始提交」
> 一次性入库。故 v0.8 之前的细节见 `程序/docs/交接说明_HANDOFF.md`；
> v0.8 起每一步都有独立 git 提交可追溯。只有 **v1.0.0 打了 git tag**（当前 HEAD）——
> 更早的里程碑代码早于现在的构建系统，不宜重建，故仅在此文档标注、不回溯打 tag。

## [未发布] — 构建产物保全

**动机**：v1.0.0 的 exe 产物在本机丢失。查因发现两处：`release/` 被 `.gitignore`
整体排除（git 无法找回），且 `build.py` 每次构建都会 `shutil.rmtree()` 同版本同
profile 的旧目录——**重建一次，上一次的产物就没了**，且无任何提示。

**改动**（只动构建与版本管理，不动程序功能，exe 内容与 v1.0.0 一致）：
- **构建脚本不再删除任何历史产物**。目标目录已存在时**默认中止**并提示改版本号；
  确要重建同一版本需显式 `--overwrite`，旧产物被**移动**到
  `release/_history/<名字>-<时间戳>/` 留档而非删除。`--clean` 也只动 `build/`。
- **新增版本台账 `release/RELEASES.md`**，由构建脚本自动维护并**纳入 git**
  （`.gitignore` 改 `release/*` 并对其单独取反——用 `release/` 会整体排除目录、
  取反不生效）。二进制仍不入库（走 GitHub Releases），但"发过哪些版本、何时、
  由哪个提交构建、多大"永久可查，本地产物丢了也能从对应提交精确重建。
- `--list` 改为同时列出当前版本与 `_history/` 中被顶替的产物，并显示来源提交。

**本次重建产物**：v1.0.0 full（428.0 MB，含 zip）与 lite（222.9 MB，含 zip），
两者均已实测启动并返回 HTTP 200。

## [1.0.0] — 2026-07-20 · 正式版

**一句话**：从"英译中 PDF 工具"成长为**多格式、多语言、有测试、可打包、已开源**的
完整文档翻译器。

**相比 v0.9 新增**：
- **多语言方向（B5）**：源 11 种（含自动检测）× 目标 10 种（中/英/日/韩/德/法/西/俄/
  葡/意）。此前只有英→中。含脚本感知断行（欧洲重音不断词、CJK 逐字、韩俄按词）、
  目标语感知的"是否已翻译"判定、按语言对的质量长度带、缓存键含语言对。
- **语言→模型推荐**：界面内联提示，选目标语后建议更合适的服务并可一键切换
  （中英用 DeepSeek、欧洲语言用 OpenAI…）。
- **PowerPoint（.pptx）支持（A3）**：文本框、表格、演讲者备注，样式保真。
- **精简版（lite）构建**：不含 OCR，体积约为完整版一半（149MB vs 298MB）。
- **首次运行引导**：未配置 Key 时给三选一引导页（DeepSeek 申请 / Ollama 免费离线 /
  离线试用），而非等点击后报错。
- **CI（GitHub Actions）**：推送自动跑测试；打 `v*.*.*` tag 自动构建 full+lite 并发布。
- 修复：build.py 的 full/lite 同版本号互相覆盖（改为 `release/v<版本>-<profile>/`）；
  中→英/日→中"原样返回"bug（`_worth_translating` 曾写死"已是中文就跳过"）。

**支持格式**：PDF · Word(.docx) · PowerPoint(.pptx) · Markdown · TXT · SRT
**支持服务**：DeepSeek · Kimi · 智谱GLM · 豆包 · OpenAI · Ollama本地 · 自定义
**测试**：179 项 pytest（CI 干净环境 168 过 + 11 跳过依赖真实论文的用例）

对应提交：`2ece5d7`(lite+引导) `732fb85`(CI) `72ac8f5`(多语言) `a063062`(PPTX)

## [0.9] — 2026-07-19 · 加固与扩展（未打 tag）

**相比 v0.8 新增**：把"能用的脚本"变成"可协作、可分发的工程"。

- **自动化测试套件（E1）**：pytest，把此前散落的手工验证沉淀为可重复回归
  （解析结构不变量、缓存、失败降级、占位符保护、跨栏配对、端到端）。
- **表格逐单元格翻译（B1）**：一格=一个翻译单元，译文不越格，表格结构不崩。
- **Word 文档（.docx）支持（A1）**：按 run 就地替换，标题层级/项目符号/表格/
  页眉页脚天然保留。
- **原生应用窗口（C2）**：pywebview，去掉黑窗口与浏览器地址栏，失败自动回退浏览器。
- **译文质量自检与定向重译（B3）**：截断/啰嗦/数字错漏/元话语/重复退化五类检查，
  命中则带着具体问题重译一次。阈值按 116 段真实语料标定，零误报。
- **Markdown / TXT / SRT 支持（A4）**：代码块、链接地址、时间轴等用占位符保护，
  绝不被改动。
- **一键打包与版本管理（C1）**：`build.py` 出 exe，免装 Python；版本唯一真源、
  自动图标、SHA256 清单、多版本按目录归档、代码签名钩子。
- **开源：采纳 AGPL-3.0**（路线 A：开源 + 增值服务）。补 `LICENSE` / `NOTICE.md`。
- **版本管理：git init + 推送私有仓库** `morning-cml/pdf-translator`。

## [0.8] — 2026-07-19 · 核心引擎（初始提交 `3b9aeed`，含 8 轮 pre-git 迭代）

首次入库时已具备的能力（此前 8 轮迭代累积，细节见交接说明）：

1. **精确回填内核**：PyMuPDF 精确抹除原文字符（深色背景零露白）+ reportlab 覆盖兜底，
   自动选择与回退。
2. **行内公式保护**：检测 → `⟦Fn⟧` 占位 → 译后校验 → 矢量回贴。
3. **共享重排引擎**：中文断行禁则、逐行避障绕图、向下扩展、自适应缩号、两端对齐。
4. **结构还原**：数据驱动分栏、标题层级（加粗+原色）、参考文献免译保排、跨栏断句缝合。
5. **扫描版 OCR**：RapidOCR 本地离线，无需显卡。
6. **可选版面分析模型**：DocLayout-YOLO（识别表格/图区/独立公式）。
7. **翻译侧**：六家服务预设、术语库、全文语境注入、持久化缓存、失败降级、成本预估、
   试译模式（只翻前 N 页）。
8. **三种输出**：纯中文 / 双语·前后页 / 双语·左右对照宽页。
9. **网页皮肤界面**：分层前端架构，皮肤=一个 CSS 文件（含海绵泡泡、清爽白两套）。
   tkinter 版冻结为备用。

> 这 9 项对应交接说明里的「第一~八轮迭代」，是本项目的技术底座。
