## 隔离现状

- PyMuPDF 的使用**已集中在** `src/pdf_writer_fitz.py`、`src/layout_model.py`、
  `src/ocr.py`（渲染位图）与 `samples/make_scanned.py`，其余模块不直接依赖；
  `pipeline.pick_backend()` 已具备"缺失即回退 reportlab"的完整路径。
- 版面模型为纯可选：删除 `models/` 下的 .onnx 文件即自动停用。
- 因此**切换到全宽松许可栈是可行的**，主要工作量在提升 reportlab 后端质量
  （路线 B / 方向四 D3）。

## 本项目自身的许可证

**已确定：AGPL-3.0（2026-07-19 采纳路线 A「开源 + 增值服务」）。**

- 许可证全文：仓库根目录 `LICENSE`（SPDX: AGPL-3.0）
- 版权与合规说明：根目录 `NOTICE.md`
- 因链接 AGPL-3.0 的 PyMuPDF 与 DocLayout-YOLO，本项目必须采用兼容的
  AGPL-3.0；这与"开源 + 增值服务"的商业模式一致。

**据此确立的红线**（后续开发必须遵守）：
1. 任何分发版本、任何联网提供的服务，都必须附带完整源代码（AGPL 第 13 条对
   网络服务同样生效）。
2. **不得**以闭源形式分发本产品。若将来确需闭源，须先完成"移除 AGPL 组件"
   的改造（见路线 B / 方向四 D3），或购买 Artifex 与 Ultralytics 的商业授权。
3. 新引入依赖前先核许可：优先选择 MIT / BSD / Apache 2.0；引入新的强 copyleft
   组件前须评估其对分发方式的影响，并更新本文件。
