# 在 Mac 上第一次打开（请先看这一页）

> 你收到的是一个**免费、开源**的小工具。因为作者没有购买苹果每年几百块的
> 「开发者认证」，macOS 第一次打开它时会弹出安全提示——**这是正常现象，
> 软件本身是安全的**。跟着下面做**一次**就好，之后就能像普通软件一样直接双击。

整个过程约 2 分钟，看你电脑弹出的是哪种提示，照对应的一步做即可：
- 弹「**无法验证开发者 / 来自身份不明的开发者**」→ 做 **第 1 步**
- 弹「**已损坏，无法打开**」→ 做 **第 2 步**

---

## 第 0 步：先把软件放好

- 收到的是 **`.zip`**：双击它，会解压出一个叫 **「PDF翻译工具」** 的图标。
- 收到的是 **`.dmg`**：双击打开，把里面的 **「PDF翻译工具」** 图标拖到
  **「应用程序」** 文件夹。

建议把「PDF翻译工具」放进 **「应用程序」** 或 **桌面**，方便下次找到。

---

## 第 1 步：提示「无法验证开发者」时（最常见）

1. 双击 **「PDF翻译工具」**。
2. 弹出提示后，**先点「完成」或「好」关掉它**（⚠️ 千万别点「移到废纸篓」）。
3. 点屏幕左上角 **苹果标志 () → 系统设置 → 隐私与安全性**。
4. 在右侧**往下滚动**，会看到一行：
   **「已阻止使用"PDF翻译工具"…」**，它旁边有个按钮 **「仍要打开」** —— 点它。
5. 再确认一次（可能要输开机密码或按指纹），软件就打开了。
6. ✅ **以后再打开就不会再问了**，直接双击即可。

> 老一点的 Mac 也可以更简单：在「PDF翻译工具」图标上 **点右键 → 打开**，
> 在弹出的小窗里再点一次 **「打开」**。

---

## 第 2 步：提示「已损坏，无法打开」时

**别担心，软件没有坏**，这只是更严格的一道拦截。用一行命令解开，只需做一次：

1. 按 **⌘（Command 键）+ 空格**，输入 **终端**，回车——打开一个黑色的「终端」窗口。
2. 在里面**照打**下面这几个字符（注意**最后留一个空格**，先别回车）：

   ```
   xattr -cr 
   ```

3. 用鼠标把 **「PDF翻译工具」的图标**（在桌面或应用程序里那个）**拖进终端窗口**
   —— 它的位置会自动补在命令后面。
4. 按 **回车**。没有任何提示，就是成功了。
5. 回去双击「PDF翻译工具」，按 **第 1 步** 的方法打开即可。

> 拖图标这一步是为了免去手打路径、避免出错。整条命令的意思只是「去掉系统给
> 下载文件加的隔离标记」，不会改动软件本身。

---

## 第 3 步：开始翻译

打开后会出现一个窗口：

1. **第一次会引导你配置翻译服务**。推荐 DeepSeek（很便宜），或用 Ollama 完全
   免费离线跑。按提示把 API Key 填进去、点一下「测试连接」即可。
2. 点 **「选择文件」** 挑一个 PDF（也支持 Word / PPT / Markdown / 字幕），点 **「开始」**。
3. 译好的文件就在**原文件旁边**，文件名带 `_translation`。

> 💡 只想先确认它能不能正常打开、暂时不想申请 API Key？在首次引导里选
> **「离线试用」**（或在设置的高级项里勾选「离线测试」），随便挑个 PDF 点开始，
> 就能看到排版效果——不会真翻译、不花钱、也不联网。

更详细的用法见随附的 **《使用说明.html》**——双击它用浏览器打开就能看，
**不用先打开软件也能读**。

---

## 这到底安不安全？

安全。它是开源软件，任何人都能查看它的全部源代码：
**github.com/morning-cml/pdf-translator**。不含广告、不含后门，你的 API Key
**只保存在你自己的电脑上**，不会上传到任何地方。

macOS 弹那些警告，唯一的原因是作者没有向苹果缴纳每年的开发者年费——
它跟软件的质量、安全都没有关系。你在网上下载的很多免费开源 Mac 小工具，
第一次打开都要这么操作一下。
