# 后续优化与扩展规划（v1.0.0 之后）

> 制定于 2026-07-20。本文件是**面向未来**的规划：只讲还没做的，按主题分组、
> 标注价值/工作量/风险/依赖，并给出推荐的推进顺序。
> 已完成的历史见 `../../CHANGELOG.md`；当前现状与遗留见 `项目总览.md`；
> 早期的方向评估与许可/竞品调研见 `未来路线图.md`（其"进度表"以本文件为准）。

**规划总原则**（承接 AGPL-3.0 路线 A「开源 + 增值服务」）：
先补已交付功能的**短板**（名实相符）→ 再做**差异化质量**（护城河）→ 再扩**格式/生态**
→ 最后落**变现**（云端/增值）。工作量记号：S=半天内 · M=1~3 天 · L=1 周以上。

## 主题一：翻译质量（护城河，最值得投入）

| 编号 | 功能 | 价值 | 工作量 | 风险/依赖 |
|---|---|---|---|---|
| Q1 | **全文术语一致性**：先扫全篇抽高频专名/术语 → 生成"本文术语表"注入每批，保证同一词全文同译 | 高 | M | 低；复用 glossary 注入通道。BabelDOC 论文列为关键改进点 |
| Q2 | **翻译记忆（TM）跨文档复用**：把 transcache 从"精确同段命中"升级为"相似段模糊命中"，跨文档省钱提一致性 | 中高 | M | 需轻量向量/最小编辑距离；注意误命中 |
| Q3 | **两遍翻译（草稿→润色）**：可选的第二遍，让模型自审并改写关键段 | 中 | S | 翻倍 token；做成开关，默认关 |
| Q4 | **邻段上下文注入**：当前只注入标题+摘要；再带前后各一段，长文连贯性更好 | 中 | S | 略增 token；对话式模型收益明显 |
| Q5 | **领域库可扩充/自动匹配**：内置多学科术语库（生医/物理/法律…），按文档自动选 | 中 | M | 需整理各领域词表；领域检测已有雏形 |
| Q6 | **图内文字翻译**（图表/截图里的英文）：对嵌入图像 OCR → 在原位叠加译文 | 中 | L | 高风险，易破坏图；做成可选，仅限清晰位图 |
| Q7 | **人工校对回写**：界面里改译文 → 记住该更正（写入 TM），下次同段直接用 | 中 | M | 依赖 Q2 的 TM 基础 |

## 明确暂缓 / 不做（附理由）

- **RTL 语言（阿拉伯语/希伯来语）**：需双向文本(bidi)重排，改动面极大、受众小——暂缓。
- **二进制混淆/防逆向**：AGPL 下须附源码，混淆无意义且可能违反许可——不做
  （详见第三方许可.md；除非将来转闭源路线 B）。
- **IR 中间表示大重构（B6）**：借鉴 BabelDOC 彻底分离版面与语义，收益大但改动
  面极大——待 Q1/F1 等沉淀、确有复杂版式瓶颈后再评估，不列入近期。

## 推荐推进顺序（下一步该做什么）

**阶段一 · 补短板 + 立信任（都是 S/M，快见效）**
1. **P1 任务队列 + P6 补失败段**：消除"一次一个任务"的明显短板。
2. **M5 开源获客 + E2 正式发版**：打 tag 发 v1.0.0，英文 README + 效果图。
3. **S3 诊断包**：为后续支持与云端排障铺路。

**阶段二 · 筑护城河（质量差异化）**
4. **Q1 全文术语一致性**：单项性价比最高的质量提升。
5. **Q4 邻段上下文** + **Q3 两遍翻译（开关）**：低成本提连贯与准确。
6. **F1 PDF 中/日文源**：补多语言的最大遗留（先备 CJK 测试样张）。

**阶段三 · 扩格式与生态**
7. **F3 Excel + F2 EPUB + F4 HTML**：三个都轻，快速扩"支持格式"清单。
8. **P3 术语库可视化编辑 + P2 拖拽**：产品打磨。

**阶段四 · 变现**
9. **M1 云端版 + M2 用量面板**：路线 A 的主要变现形态（大工程，独立立项）。

> 若目标是**尽快变现**：阶段一后可直接跳 M1 云端版。
> 若目标是**做强产品/开源影响力**：按阶段一→二→三顺序，护城河优先。
