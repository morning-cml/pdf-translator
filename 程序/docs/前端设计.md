# 网页前端架构设计（2026-07-18 制定）

> 目标：把界面从 tkinter 迁移为**本地网页应用**，支持"皮肤级"多主题切换，
> 模块独立、可扩展，为未来商业化（在线服务/付费皮肤/账号体系）预留演进路径。

## 0. 技术选型与理由

| 决策 | 选择 | 理由 |
|---|---|---|
| 前端形态 | 本地网页（浏览器打开 127.0.0.1） | 皮肤=CSS，表现力无上限；未来可无缝演进为在线 SaaS |
| 后端 | Python 标准库 http.server | **零新依赖**，延续"双击即用"；翻译内核 100% 复用 |
| 前端框架 | 无（原生 ES Modules） | 零构建、零 node 依赖；模块边界靠文件与约定 |
| 通信 | JSON REST + 轮询进度 | 标准库即可实现，简单可靠 |
| 安全 | 只绑定 127.0.0.1 | 不对外网暴露；含 Key 的接口仅本机可达 |

## 1. 总体分层（自下而上）

```
┌────────────────────────────────────────────────────────┐
│ L4 皮肤层  web/themes/<id>/theme.css （+ 自带 SVG 资源）  │  ← 只写 CSS 变量与装饰
├────────────────────────────────────────────────────────┤
│ L3 组件层  web/js/components.js（文件篮/设置面板/进度卡…）│  ← 只操作 DOM + 调 store/api
├────────────────────────────────────────────────────────┤
│ L2 状态层  web/js/store.js（发布-订阅式全局状态）          │  ← 唯一数据源
│    通信层  web/js/api.js（fetch 封装，全部接口在此）       │
├────────────────────────────────────────────────────────┤
│ L1 服务层  webui.py（HTTP API + 静态文件 + 任务线程）      │  ← 唯一与翻译内核接触的层
├────────────────────────────────────────────────────────┤
│ L0 翻译内核  src/pipeline 等（本轮零改动）                 │
└────────────────────────────────────────────────────────┘
```

**独立性规则**（保证可扩展的核心约束）：
1. 皮肤层**只允许**定义 CSS 变量与装饰性规则，禁止改布局结构与 JS——
   任何主题坏了都不影响功能。
2. 组件层不直接 fetch，一律经 `api.js`；不直接互相调用，经 `store.js` 通信。
3. 服务层是翻译内核唯一入口——将来换成云端后端时，前端零改动
   （api.js 里改一个 BASE_URL 即可）。

## 2. 皮肤（主题）契约

每个皮肤 = `web/themes/<id>/` 一个文件夹：

```
web/themes/
├─ themes.json            # 注册表：[{id, name, emoji, dark}]
├─ clean/theme.css        # 极简白（默认兜底，证明契约完整性）
└─ sponge/theme.css       # 海绵泡泡（首个卡通主题）
```

`theme.css` 必须给出 **全部设计令牌**（CSS 变量，:root 作用域）：

| 令牌 | 含义 |
|---|---|
| `--c-bg / --c-bg-deco` | 页面背景色 / 装饰性背景（渐变、图案） |
| `--c-surface / --c-surface-2` | 卡片面 / 次级面 |
| `--c-text / --c-text-muted` | 正文 / 弱化文字 |
| `--c-primary / --c-primary-text` | 主按钮底/字 |
| `--c-accent / --c-danger / --c-ok` | 强调 / 危险 / 成功 |
| `--c-border` | 描边 |
| `--radius / --radius-lg` | 圆角（卡通主题可以很大） |
| `--shadow` | 卡片阴影 |
| `--font-ui / --font-display` | 正文字体 / 标题字体 |

装饰挂点（可选实现）：`body::before/::after`（背景装饰层）、
`.app-header .mascot`（吉祥物）、`.progress-card.running .mascot`（工作状态动画）、
`.btn-primary:hover` 弹跳等。**新增皮肤三步**：建文件夹 → 写 theme.css →
在 themes.json 加一行。运行时由 `theme.js` 切换 `<link id="theme-css">` 并写入
localStorage 记忆。

> 版权红线：卡通主题一律使用**原创绘制**的 CSS/SVG 元素（气泡、海绵质感、
> 菠萝剪影等风格化表达），不得使用受版权保护的现成角色形象与名称
> （如海绵宝宝本体）。商用前如需真 IP，须取得授权。

## 3. 组件清单（components.js 内按节导出）

| 组件 | 职责 | 依赖 |
|---|---|---|
| AppShell | 页头（标题/吉祥物/主题切换器） | theme.js |
| FileBasket | 文件篮：本机选择（原生对话框）/ 手动粘贴路径 / 列表增删 | api.pick |
| SettingsPanel | 服务/模型/Key/领域/输出模式/试译/并发/后端/三开关；折叠分组"基础/高级" | store |
| RunBar | 开始/取消按钮 + 状态一行字 | api.jobs |
| ProgressCard | 进度条 + 阶段文案 + 吉祥物状态（皮肤可做动画） | store |
| LogView | 滚动日志（费用预估/缓存命中/失败降级都在这里出现） | store |
| ResultsList | 每个完成文件：打开 PDF / 打开文件夹 | api.open |
| Toast | 轻提示（保存成功/错误） | — |

## 4. 服务层 API（webui.py）

| 方法与路径 | 作用 |
|---|---|
| `GET /` `/web/*` | 静态文件（index.html、css、js、themes） |
| `GET /api/config` | 读当前配置（含 Key，仅本机） |
| `POST /api/config` | 保存配置（写 config.json） |
| `GET /api/themes` | 皮肤注册表 |
| `POST /api/pick` | 弹本机文件选择框，返回所选 PDF 路径列表 |
| `POST /api/test` | 测试翻译服务连通性 |
| `POST /api/jobs` | 建翻译任务 {files, overrides}；后台线程执行，返回 job_id |
| `GET /api/jobs/{id}` | 轮询：{status, percent, message, log[], outputs[]} |
| `POST /api/jobs/{id}/cancel` | 取消任务 |
| `POST /api/open` | 用系统默认程序打开产物 PDF / 所在文件夹 |

任务模型：单进程内存字典 `JOBS[id]`；一次一个活动任务（与 tkinter 版一致），
将来要并发/多用户时把它换成队列即可，接口不变。

## 5. 未来扩展路线（谋利方向的预留点）

- **付费皮肤**：themes.json 增加 `paid: true` + 服务层校验解锁码——皮肤契约
  不变，只在 theme.js 的加载入口加一道闸。
- **云端化**：webui.py 的 API 语义即云端 API 草案；上云时前端仅改 BASE_URL，
  增加登录页与用量计费页（新组件，不动现有组件）。
- **多语言界面**：组件层文案已集中在各组件顶部常量，抽 `i18n.js` 即可。
- **移动端**：base.css 已用响应式单列布局，小屏天然可用。
- tkinter 旧界面保留（启动PDF翻译.bat），网页版稳定后可替换默认入口。
